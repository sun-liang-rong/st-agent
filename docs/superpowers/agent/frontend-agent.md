# 前端 Agent 任务分工

## 职责边界

- 所有页面组件、通用组件的实现
- 样式与响应式布局
- Pinia Store 状态管理
- API 服务层调用
- 路由配置
- 不涉及后端 API、数据库、服务层

---

## P0 阶段（第 1-2 周）

### T-F01: 对话中生成图片 — 前端

| 项目 | 内容 |
|------|------|
| 优先级 | P0 |
| 预计工时 | 2.5 天 |
| 依赖 | T-B02（后端 SSE 事件流就绪） |
| 交付物 | 对话页图片生成交互 |

**任务清单**：
- [ ] 输入框区域改造：
  - 添加风格选择器（胶囊按钮，默认折叠，点击展开）
  - 添加比例选择器（胶囊按钮，默认折叠，点击展开）
  - 选择器使用 `<Teleport to="body">` 挂载，`z-index: 9999`
  - 选中风格/比例后在输入框底部显示标签，可清除
  - 无选择时走纯对话逻辑，有选择时触发图片生成
- [ ] SSE 事件解析扩展（`api.ts`）：
  - `image_intent` — 显示"正在生成图片..."提示
  - `image_progress` — 显示进度（构思/生成/下载）
  - `image_result` — 在消息流中插入图片卡片
- [ ] 图片卡片组件（在消息气泡内）：
  - 圆角卡片，显示生成的图片
  - hover 显示操作按钮：查看大图 / 下载 / 重新生成
  - 图片加载失败显示占位图
- [ ] 大图预览：点击图片卡片弹出全屏预览
- [ ] 与现有流式对话逻辑兼容，不破坏纯文本消息渲染

## 通用规范

- 所有新组件使用 Vue 3 `<script setup lang="ts">` 语法
- 样式遵循 TailwindCSS + amber/orange 设计语言
- 新页面必须注册路由，需要认证的添加 `meta.requiresAuth`
- API 调用统一通过 `api.ts`，不直接在组件中写 axios/fetch
- 状态管理优先使用 Pinia Store，避免组件内大量响应式状态
- 移动端适配使用 TailwindCSS 断点：`sm:768px`, `md:1024px`, `lg:1280px`
